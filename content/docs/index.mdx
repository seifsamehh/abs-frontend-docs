## Our Tech Stack

| Technology         | Version | Purpose                                     |
| ------------------ | ------- | ------------------------------------------- |
| **React**          | 19.x    | UI library with Server Components & Actions |
| **Next.js**        | 16.x    | Full-stack framework with App Router        |
| **TypeScript**     | 5.9.x   | Type-safe JavaScript                        |
| **Tailwind CSS**   | 4.x     | CSS-first utility framework                 |
| **TanStack Query** | 5.x     | Server state management & caching           |
| **Zustand**        | 5.x     | Simple client state management              |
| **shadcn/ui**      | Latest  | Accessible component library                |
| **Vitest**         | 2.x     | Unit & integration testing                  |
| **Playwright**     | 1.x     | E2E testing                                 |

## Core Principles

<Accordions>
  <Accordion title="Performance First">
    We build fast, responsive applications. Every component, every route, every bundle is optimized for speed. We measure Core Web Vitals and set performance budgets.
  </Accordion>
  <Accordion title="Developer Experience">
    Great tools make great engineers. We invest in TypeScript, editor configs, linting, and automation so you can focus on building features instead of fighting tooling.
  </Accordion>
  <Accordion title="Maintainability">
    Clean, readable, and well-tested code lasts. We prefer explicit over clever, boring over fancy. Every PR gets reviewed. Every pattern is documented.
  </Accordion>
  <Accordion title="Accessibility">
    We build for everyone. ARIA attributes, keyboard navigation, screen reader support, and WCAG compliance are non-negotiable.
  </Accordion>
  <Accordion title="Type Safety">
    Catch errors at compile time, not runtime. TypeScript strict mode, Zod for runtime validation, and type-safe APIs end-to-end.
  </Accordion>
</Accordions>

[Read our full engineering philosophy →](/docs/architecture/philosophy)
