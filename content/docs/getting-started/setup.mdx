## Development Environment Setup

### 1. Clone the Repository

```bash
# Clone the main repository
git clone https://github.com/abs/frontend-docs.git
cd frontend-docs

# Install dependencies
npm install
```

Example `.env.local`:

```bash
NEXT_PUBLIC_API_URL=http://localhost:3000
```

### 3. Start Development Server

```bash
# Start the development server
npm run dev
```

Your application should now be running at `http://localhost:3000`!

## Making Your First Contribution

### 1. Find an Issue

Start with beginner-friendly issues:

- Look for `good first issue` labels
- Check the documentation for missing content
- Fix typos or improve examples
- Add missing type definitions

### 2. Create a Branch

```bash
# Create and switch to a new branch
git checkout -b feature/add-component-example

# Or for bug fixes
git checkout -b fix/typo-in-readme
```

### 3. Make Your Changes

Follow our coding standards:

```typescript
// Example: Adding a new component documentation
---
title: Button Component
description: Reusable button component with variants and states
---

import { Button } from '@/components/ui/button';

# Button Component

The Button component provides a consistent way to trigger actions.

## Basic Usage

<Button>Click me</Button>

## Variants

<Button variant="primary">Primary</Button>
<Button variant="secondary">Secondary</Button>
<Button variant="outline">Outline</Button>

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| variant | string | 'primary' | Button style variant |
| size | string | 'md' | Button size |
| disabled | boolean | false | Disabled state |
```

### 4. Test Your Changes

```bash
# Run the development server
npm dev

# Check for TypeScript errors
npm type-check

# Run linting
npm lint

# Format code
npm format
```

### 5. Commit and Push

```bash
# Stage your changes
git add .

# Commit with conventional commit format
git commit -m "docs: add button component documentation"

# Push to your branch
git push origin feature/add-component-example
```

### 6. Create Pull Request

1. Go to the repository on GitHub
2. Click "New Pull Request"
3. Select your branch
4. Fill out the PR template
5. Request review from team members

## Development Workflow

### Daily Workflow

1. **Start of Day**

   ```bash
   # Pull latest changes
   git checkout main
   git pull origin main

   # Start development server
   npm dev
   ```

2. **Working on Features**

   ```bash
   # Create feature branch
   git checkout -b feature/new-feature

   # Make changes and test
   # Commit regularly with good messages
   git add .
   git commit -m "feat: implement new feature"
   ```

3. **End of Day**

   ```bash
   # Push your work
   git push origin feature/new-feature

   # Create PR if ready for review
   ```

### Onboarding Checklist

Use this checklist for your first week:

- [ ] **Day 1**: Environment setup and repository access
- [ ] **Day 1**: Complete VS Code configuration
- [ ] **Day 2**: Read through main documentation sections
- [ ] **Day 2**: Join team Whatsapp channels
- [ ] **Day 3**: Attend team standup and introduce yourself
- [ ] **Day 4**: Set up 1-on-1 with your mentor
- [ ] **Day 5**: Complete a "good first issue"
- [ ] **Week 1**: Schedule code review session
- [ ] **Week 1**: Join learning group sessions

### Next Steps

After completing the basics:

1. **Choose a Learning Path**: Focus on areas you want to improve
2. **Join a Project**: Contribute to active development projects
3. **Attend Team Events**: Participate in code reviews and tech talks
4. **Share Knowledge**: Write about what you learn
5. **Mentor Others**: Help new team members when you're ready
