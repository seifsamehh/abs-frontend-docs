# Next Steps

  Now that you're set up and made your first contribution, here's how to
  continue growing as a frontend developer at ABS.

## Learning Paths by Role

<Tabs items={["Junior Developer", "Mid-Level Developer", "Senior Developer"]}>
  <Tab value="Junior Developer">
    ### First 3 Months

    **Focus Areas:**
    - Master React fundamentals
    - Learn TypeScript basics
    - Understand our codebase structure
    - Write your first tests

    **Recommended Reading:**
    1. [Component Building Guide](/docs/guides/components)
    2. [Code Standards](/docs/reference/code-standards)
    3. [Testing Guide](/docs/guides/testing)
    4. [Architecture Overview](/docs/reference/architecture)

    **Projects to Try:**
    - Build 3-5 UI components
    - Add tests to existing components
    - Fix 5+ bugs from issue tracker

    **Mentorship:**
    - Pair program with senior developers
    - Request code review feedback
    - Ask questions in code reviews

</Tab>
  <Tab value="Mid-Level Developer">
    ### First 3 Months

    **Focus Areas:**
    - Master state management patterns
    - Deep dive into Next.js App Router
    - Learn performance optimization
    - Improve testing strategies

    **Recommended Reading:**
    1. [Data Fetching Guide](/docs/guides/data-fetching)
    2. [State Management Guide](/docs/guides/state-management)
    3. [Authentication Guide](/docs/guides/authentication)
    4. [Development Philosophy](/docs/learning/philosophy)

    **Projects to Try:**
    - Lead a feature implementation
    - Improve an existing feature's performance
    - Mentor a junior developer

    **Growth:**
    - Propose technical improvements
    - Write documentation
    - Present in team meetings

</Tab>
  <Tab value="Senior Developer">
    ### First 3 Months

    **Focus Areas:**
    - Understand system architecture
    - Identify improvement opportunities
    - Build relationships with stakeholders
    - Establish technical direction

    **Recommended Reading:**
    1. [Architecture Reference](/docs/reference/architecture)
    2. [Security Guidelines](/docs/reference/security)
    3. [AI-Assisted Development](/docs/guides/ai-assisted-development)
    4. [Deployment Guide](/docs/guides/deployment)

    **Projects to Try:**
    - Audit and improve existing architecture
    - Implement cross-cutting concerns
    - Establish or improve patterns

    **Leadership:**
    - Lead technical discussions
    - Mentor team members
    - Drive documentation improvements

</Tab>
</Tabs>

## Skill Development Checklist

### Core Skills

- [ ] **React** - Components, hooks, patterns
- [ ] **TypeScript** - Types, interfaces, generics
- [ ] **Next.js** - App Router, server components, routing
- [ ] **Tailwind CSS** - Utility classes, responsive design
- [ ] **Testing** - Unit tests, integration tests

### Advanced Skills

- [ ] **State Management** - TanStack Query, Redux Toolkit
- [ ] **Forms** - React Hook Form, Zod validation
- [ ] **Authentication** - NextAuth.js patterns
- [ ] **Performance** - Optimization, profiling
- [ ] **Accessibility** - WCAG, screen readers

### Expert Skills

- [ ] **Architecture** - System design, patterns
- [ ] **DevOps** - CI/CD, deployment strategies
- [ ] **Security** - OWASP, secure coding
- [ ] **Leadership** - Mentoring, technical direction

## Weekly Growth Activities

### Every Week

1. **Read** - One article or documentation section
2. **Build** - One small feature or improvement
3. **Review** - Two pull requests from teammates
4. **Share** - One thing you learned with the team

### Monthly

1. **Reflect** - What did you learn? What's next?
2. **Document** - Add or improve one documentation page
3. **Mentor** - Help a teammate with something
4. **Explore** - Try a new tool or technique

## Questions to Ask

**Ask your team lead:**

- What are the team's current priorities?
- What would have the most impact?
- What should I focus on learning?

**Ask senior developers:**

- Why was this architectural decision made?
- What patterns should I follow?
- What mistakes should I avoid?

**Ask yourself:**

- What am I struggling with?
- What do I want to learn next?
- How can I help the team?
