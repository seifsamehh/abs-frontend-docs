```
# Frontend Development Rules for AI Tools

## Project Structure
- Use feature-based organization in `features/` directory
- Keep `app/` routes minimal, delegate logic to features
- No cross-feature imports - use `index.ts` re-exports
- No logic in UI components - use hooks/services

## Naming Conventions
- Components: PascalCase (UserCard.tsx)
- Hooks: useCamelCase
- Files: kebab-case (user-profile.tsx)
- Types: PascalCase + suffix (UserType, UserProps)
- Constants: UPPER_SNAKE_CASE

## Code Standards
- TypeScript strict mode - no `any` types
- Zod validation for all external inputs
- Tailwind + shadcn/ui for styling
- Absolute imports (@/features/...)
- Server components by default, "use client" only when needed

## Testing
- Jest + React Testing Library for unit tests
- Include tests with all AI-generated code
- Minimum 80% coverage for business logic
- Accessibility testing with axe

## Performance
- Use next/image and next/font
- Avoid unnecessary re-renders
- Animate transform + opacity only
- Use next/dynamic for large modules

## Security
- Validate all external inputs with Zod
- Never rely on client-only auth checks
- Sanitize any HTML content
- Use secure token storage (httpOnly cookies)

## Accessibility
- Semantic HTML elements
- Proper ARIA attributes
- Keyboard navigation support
- Focus management
- Screen reader compatibility

## Code Quality
- Single responsibility principle
- Small, composable components
- Early returns for better readability
- Proper error handling
- Comprehensive documentation

## Required Elements
Every AI-generated code must include:
- TypeScript types and interfaces
- Zod validation schemas
- Comprehensive unit tests
- TSDoc documentation
- Accessibility attributes
- Error handling
- Performance optimizations
```
