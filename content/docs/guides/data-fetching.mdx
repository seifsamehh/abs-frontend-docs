# 📡 Data Fetching

<Callout type="info">
  This guide walks you through fetching, caching, and mutating data using
  **TanStack Query v5** for client-side operations and **Next.js Server Actions**
  for server-side mutations. It starts with the basics and progresses to
  production-ready patterns including Zod validation, optimistic updates, and
  React 19 integration.
</Callout>

## 🛠️ Our Data Stack

| Library               | Purpose                                  |
| --------------------- | ---------------------------------------- |
| **TanStack Query v5** | Client-side caching & server state       |
| **Server Actions**    | Server-side mutations & form submissions |
| **Native Fetch**      | Built-in HTTP client                     |
| **Zod**               | Runtime request & response validation    |

<Callout type="warn">
  **TanStack Query v5 breaking changes from v4:**
  - `cacheTime` renamed to `gcTime`
  - `isLoading` replaced with `isPending` for initial load
  - `isInitialLoading` removed — use `isPending && isFetching`
  - `status: 'loading'` replaced with `status: 'pending'`

  See [ADR-001: TanStack Query](/docs/architecture/decisions/adr-001-tanstack-query) for why we chose this library.
</Callout>

```bash
pnpm add @tanstack/react-query @tanstack/react-query-devtools zod
```

</Step>

<Step>
### Configure the Query Client

Create a shared `QueryClient` with sensible defaults. This instance controls caching behaviour, retry logic, and refetch strategies across the entire app.

```tsx
// src/lib/query-client.ts
import { QueryClient } from "@tanstack/react-query";

export const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 60 * 1000,        // Data is "fresh" for 1 minute
      gcTime: 5 * 60 * 1000,       // Keep unused data in cache for 5 minutes
      retry: 3,                     // Retry failed requests 3 times
      refetchOnWindowFocus: false,  // Don't refetch every time the tab regains focus
    },
  },
});
```

<Callout type="tip">
  **`staleTime` vs `gcTime`:** Fresh data is never refetched in the background.
  Once data becomes *stale* (after `staleTime`), TanStack Query will refetch it
  next time the query is accessed. `gcTime` controls how long *unused* stale
  data stays in the cache before being garbage-collected.
</Callout>

</Step>

## 📥 Basic Data Fetching with `useQuery`

The simplest way to load data is the `useQuery` hook. It handles loading states, errors, caching, and background refetches automatically.

### Define a type and fetch function

```tsx
// src/types/user.ts
export interface User {
  id: number;
  name: string;
  email: string;
  company: {
    name: string;
  };
}
```

```tsx
// src/lib/api.ts
import type { User } from "@/types/user";

export async function fetchUsers(): Promise<User[]> {
  const response = await fetch("https://jsonplaceholder.typicode.com/users");

  if (!response.ok) {
    throw new Error("Failed to fetch users");
  }

  return response.json();
}
```

<Callout type="note">
  Always throw on non-OK responses. TanStack Query catches thrown errors and
  surfaces them through the `error` property.
</Callout>

### Build a `UserList` component

```tsx
// src/components/UserList.tsx
"use client";

import { useQuery } from "@tanstack/react-query";
import { fetchUsers } from "@/lib/api";

export function UserList() {
  const {
    data: users,
    isPending,
    isError,
    error,
    refetch,
    isFetching,
  } = useQuery({
    queryKey: ["users"],
    queryFn: fetchUsers,
    staleTime: 5 * 60 * 1000, // fresh for 5 minutes
  });

  if (isPending) {
    return (
      <div className="p-4 flex items-center gap-2">
        <div className="animate-spin h-5 w-5 border-2 border-primary border-t-transparent rounded-full" />
        Loading users...
      </div>
    );
  }

  if (isError) {
    return (
      <div className="p-4 bg-red-50 text-red-500 rounded-lg">
        Error: {error.message}
        <button onClick={() => refetch()} className="ml-2 underline">
          Try again
        </button>
      </div>
    );
  }

  if (!users?.length) {
    return <div className="p-4 text-gray-500">No users found.</div>;
  }

  return (
    <div className="p-4">
      <div className="flex items-center justify-between mb-4">
        <h2 className="text-2xl font-bold">Users ({users.length})</h2>
        <button
          onClick={() => refetch()}
          disabled={isFetching}
          className="px-4 py-2 bg-primary text-white rounded-md hover:bg-primary/90 disabled:opacity-50"
        >
          {isFetching ? "Refreshing..." : "Refresh"}
        </button>
      </div>
      <ul className="space-y-2">
        {users.map((user) => (
          <li
            key={user.id}
            className="p-4 bg-white rounded-lg shadow hover:shadow-md transition-shadow"
          >
            <p className="font-medium text-gray-900">{user.name}</p>
            <p className="text-sm text-gray-600">{user.email}</p>
            <p className="text-xs text-gray-500 mt-1">{user.company.name}</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

**Key concepts:**

- **`queryKey`** — unique cache identifier. Queries with the same key share cached data.
- **`queryFn`** — the async function that fetches data.
- **`isPending`** — `true` during the initial load (v5 replaces the old `isLoading`).
- **`isFetching`** — `true` during any fetch, including background refetches.
- **`refetch`** — call this to manually trigger a refetch.

## 🔑 Query Key Factory & Zod Validation

As your app grows, query keys can become inconsistent. A **query key factory** keeps them organised and type-safe. Pair this with **Zod** to validate API responses at runtime.

```tsx
// src/hooks/use-users.ts
import { useQuery } from "@tanstack/react-query";
import { z } from "zod";

// --- Zod schema as the single source of truth ---
const userSchema = z.object({
  id: z.string(),
  name: z.string(),
  email: z.string().email(),
  role: z.enum(["admin", "user", "guest"]),
});

const usersResponseSchema = z.array(userSchema);

export type User = z.infer<typeof userSchema>;

// --- Query key factory (recommended pattern) ---
export const userKeys = {
  all: ["users"] as const,
  lists: () => [...userKeys.all, "list"] as const,
  list: (filters: string) => [...userKeys.lists(), filters] as const,
  details: () => [...userKeys.all, "detail"] as const,
  detail: (id: string) => [...userKeys.details(), id] as const,
};

// --- Fetch + validate ---
async function fetchUsers(): Promise<User[]> {
  const response = await fetch("/api/users");
  if (!response.ok) throw new Error("Failed to fetch users");
  const json = await response.json();
  return usersResponseSchema.parse(json); // throws ZodError if shape is wrong
}

// --- Custom hook ---
export function useUsers() {
  return useQuery({
    queryKey: userKeys.lists(),
    queryFn: fetchUsers,
  });
}
```

<Callout type="tip">
  The query key factory pattern lets you invalidate at different granularities:
  `userKeys.all` invalidates everything user-related, `userKeys.lists()`
  invalidates only list queries, and `userKeys.detail(id)` targets a single
  record.
</Callout>

## ✍️ Mutations with `useMutation`

Mutations handle create, update, and delete operations. After a mutation succeeds you typically invalidate related queries so the UI stays in sync.

```tsx
// src/hooks/use-create-user.ts
import { useMutation, useQueryClient } from "@tanstack/react-query";
import { z } from "zod";
import { userKeys, type User } from "./use-users";

const createUserSchema = z.object({
  name: z.string().min(2),
  email: z.string().email(),
  role: z.enum(["admin", "user", "guest"]),
});

type CreateUserData = z.infer<typeof createUserSchema>;

async function createUser(data: CreateUserData): Promise<User> {
  const validated = createUserSchema.parse(data);
  const response = await fetch("/api/users", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(validated),
  });
  if (!response.ok) throw new Error("Failed to create user");
  return response.json();
}

export function useCreateUser() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: createUser,
    onSuccess: () => {
      // Invalidate the user list so it refetches with the new record
      queryClient.invalidateQueries({ queryKey: userKeys.lists() });
    },
    onError: (error) => {
      console.error("Failed to create user:", error);
    },
  });
}
```

### Using the mutation in a component

```tsx
// src/components/CreateUserForm.tsx
"use client";

import { useCreateUser } from "@/hooks/use-create-user";

export function CreateUserForm() {
  const createUser = useCreateUser();

  async function onSubmit(formData: FormData) {
    const data = {
      name: formData.get("name") as string,
      email: formData.get("email") as string,
      role: "user" as const,
    };

    try {
      await createUser.mutateAsync(data);
      toast.success("User created successfully");
    } catch {
      toast.error("Failed to create user");
    }
  }

  return (
    <form action={onSubmit} className="space-y-4">
      <input name="name" placeholder="Name" required className="w-full rounded border p-2" />
      <input name="email" type="email" placeholder="Email" required className="w-full rounded border p-2" />
      <button
        type="submit"
        disabled={createUser.isPending}
        className="rounded bg-primary px-4 py-2 text-white disabled:opacity-50"
      >
        {createUser.isPending ? "Creating..." : "Create User"}
      </button>
    </form>
  );
}
```

## ⚡ Server Actions (Next.js)

<Callout type="info">
  Server Actions are async functions that run on the server. Use them for form
  submissions, database writes, and mutations that benefit from server-side
  validation — especially when you don't need client-side cache management.
</Callout>

### When to use what?

| Use Case                                      | Solution                          |
| --------------------------------------------- | --------------------------------- |
| **Fetching & caching data**                   | TanStack Query                    |
| **Complex mutations with optimistic updates** | TanStack Query mutations          |
| **Simple form submissions**                   | Server Actions                    |
| **Database operations**                       | Server Actions                    |
| **File uploads**                              | Server Actions                    |
| **Actions needing revalidation**              | Server Actions + `revalidatePath` |

### Basic Server Action with Zod

```tsx
// src/app/actions/user-actions.ts
"use server";

import { z } from "zod";
import { revalidatePath } from "next/cache";

const createUserSchema = z.object({
  name: z.string().min(2, "Name must be at least 2 characters"),
  email: z.string().email("Invalid email address"),
});

export async function createUser(prevState: unknown, formData: FormData) {
  const rawData = {
    name: formData.get("name"),
    email: formData.get("email"),
  };

  // Validate input
  const result = createUserSchema.safeParse(rawData);
  if (!result.success) {
    return { error: result.error.flatten().fieldErrors, success: false };
  }

  // Perform database operation
  const user = await db.user.create({ data: result.data });

  // Revalidate the users list page
  revalidatePath("/users");

  return { success: true, user };
}
```

### Using with `useActionState` (React 19)

The `useActionState` hook from React 19 replaces the older `useFormState` pattern. It returns the current state, a bound action, and a pending flag — all in one call.

```tsx
// src/components/CreateUserServerForm.tsx
"use client";

import { useActionState } from "react";
import { createUser } from "@/app/actions/user-actions";

const initialState = { error: null as Record<string, string[]> | null, success: false };

export function CreateUserServerForm() {
  const [state, formAction, isPending] = useActionState(createUser, initialState);

  return (
    <form action={formAction} className="space-y-4">
      <div>
        <label htmlFor="name">Name</label>
        <input
          id="name"
          name="name"
          required
          className="w-full rounded border p-2"
        />
        {state.error?.name && (
          <p className="text-sm text-red-500">{state.error.name}</p>
        )}
      </div>

      <div>
        <label htmlFor="email">Email</label>
        <input
          id="email"
          name="email"
          type="email"
          required
          className="w-full rounded border p-2"
        />
        {state.error?.email && (
          <p className="text-sm text-red-500">{state.error.email}</p>
        )}
      </div>

      <button
        type="submit"
        disabled={isPending}
        className="rounded bg-blue-500 px-4 py-2 text-white disabled:opacity-50"
      >
        {isPending ? "Creating..." : "Create User"}
      </button>

      {state.success && (
        <p className="text-green-500">User created successfully!</p>
      )}
    </form>
  );
}
```

### Type-safe Action Wrapper

For consistent error handling across all server actions, create a reusable wrapper:

```tsx
// src/lib/safe-action.ts
"use server";

import { z } from "zod";

type ActionState<T> = {
  success: boolean;
  data?: T;
  error?: string;
  fieldErrors?: Record<string, string[]>;
};

export async function safeAction<T>(
  schema: z.ZodSchema,
  formData: FormData,
  action: (data: z.infer<typeof schema>) => Promise<T>
): Promise<ActionState<T>> {
  const rawData = Object.fromEntries(formData);
  const result = schema.safeParse(rawData);

  if (!result.success) {
    return {
      success: false,
      fieldErrors: result.error.flatten().fieldErrors,
    };
  }

  try {
    const data = await action(result.data);
    return { success: true, data };
  } catch (error) {
    return {
      success: false,
      error: error instanceof Error ? error.message : "Something went wrong",
    };
  }
}
```

### 📄 Pagination with `useInfiniteQuery`

For lists that load more data as the user scrolls or clicks "Load More":

```tsx
// src/hooks/use-infinite-users.ts
import { useInfiniteQuery } from "@tanstack/react-query";
import { userKeys, type User } from "./use-users";

interface PaginatedResponse<T> {
  data: T[];
  nextCursor: string | null;
  hasMore: boolean;
}

export function useInfiniteUsers() {
  return useInfiniteQuery({
    queryKey: userKeys.lists(),
    queryFn: async ({ pageParam }) => {
      const response = await fetch(`/api/users?cursor=${pageParam ?? ""}&limit=20`);
      if (!response.ok) throw new Error("Failed to fetch users");
      return response.json() as Promise<PaginatedResponse<User>>;
    },
    getNextPageParam: (lastPage) =>
      lastPage.hasMore ? lastPage.nextCursor : undefined,
    initialPageParam: null as string | null,
  });
}
```

```tsx
// src/components/InfiniteUserList.tsx
"use client";

import { useInfiniteUsers } from "@/hooks/use-infinite-users";

export function InfiniteUserList() {
```

### ⚡ Optimistic Updates

Show changes instantly and roll back if the server rejects the mutation:

```tsx
// src/hooks/use-update-user.ts
import { useMutation, useQueryClient } from "@tanstack/react-query";
import { userKeys, type User } from "./use-users";

export function useUpdateUser() {
  const queryClient = useQueryClient();

  return useMutation({
    mutationFn: async (updatedUser: Partial<User> & { id: string }) => {
      const res = await fetch(`/api/users/${updatedUser.id}`, {
        method: "PATCH",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(updatedUser),
      });
      if (!res.ok) throw new Error("Failed to update user");
      return res.json() as Promise<User>;
    },

    onMutate: async (updatedUser) => {
      // 1. Cancel in-flight refetches so they don't overwrite our optimistic update
      await queryClient.cancelQueries({
        queryKey: userKeys.detail(updatedUser.id),
      });

      // 2. Snapshot current value for rollback
      const previousUser = queryClient.getQueryData<User>(
        userKeys.detail(updatedUser.id)
      );

      // 3. Optimistically update the cache
      queryClient.setQueryData<User>(
        userKeys.detail(updatedUser.id),
        (old) => (old ? { ...old, ...updatedUser } : old)
      );

      return { previousUser };
    },

    onError: (_err, updatedUser, context) => {
      // Rollback to the snapshot on error
      if (context?.previousUser) {
        queryClient.setQueryData(
          userKeys.detail(updatedUser.id),
          context.previousUser
        );
      }
    },

    onSettled: (_data, _error, variables) => {
      // Always refetch after mutation to guarantee server/client consistency
      queryClient.invalidateQueries({
        queryKey: userKeys.detail(variables.id),
      });
    },
  });
}
```

## ⚠️ Error Handling

### Global API Client

Centralise HTTP logic and error handling in a shared API module:

```tsx
// src/lib/api-client.ts
const BASE_URL = process.env.NEXT_PUBLIC_API_URL;

export class ApiError extends Error {
  constructor(
    public status: number,
    message: string,
  ) {
    super(message);
    this.name = "ApiError";
  }
}

async function handleResponse<T>(response: Response): Promise<T> {
  if (!response.ok) {
    if (response.status === 401) {
      window.location.href = "/login";
    }
    throw new ApiError(response.status, response.statusText);
  }
  return response.json();
}

export const api = {
  get: <T,>(endpoint: string, init?: RequestInit) =>
    fetch(`${BASE_URL}${endpoint}`, {
      ...init,
      method: "GET",
      headers: { "Content-Type": "application/json", ...init?.headers },
    }).then(handleResponse<T>),

  post: <T,>(endpoint: string, data?: unknown, init?: RequestInit) =>
    fetch(`${BASE_URL}${endpoint}`, {
      ...init,
      method: "POST",
      headers: { "Content-Type": "application/json", ...init?.headers },
      body: data ? JSON.stringify(data) : undefined,
    }).then(handleResponse<T>),

  patch: <T,>(endpoint: string, data?: unknown, init?: RequestInit) =>
    fetch(`${BASE_URL}${endpoint}`, {
      ...init,
      method: "PATCH",
      headers: { "Content-Type": "application/json", ...init?.headers },
      body: data ? JSON.stringify(data) : undefined,
    }).then(handleResponse<T>),

  delete: <T,>(endpoint: string, init?: RequestInit) =>
    fetch(`${BASE_URL}${endpoint}`, {
      ...init,
      method: "DELETE",
      headers: { "Content-Type": "application/json", ...init?.headers },
    }).then(handleResponse<T>),
};
```

### Query Error Boundary

Wrap query-dependent components in an error boundary so a single failed request doesn't crash the whole page:

```tsx
// src/components/QueryBoundary.tsx
"use client";

import { QueryErrorResetBoundary } from "@tanstack/react-query";
import { ErrorBoundary } from "react-error-boundary";

export function QueryBoundary({ children }: { children: React.ReactNode }) {
  return (
    <QueryErrorResetBoundary>
      {({ reset }) => (
        <ErrorBoundary
          onReset={reset}
          fallbackRender={({ error, resetErrorBoundary }) => (
            <div className="p-6 bg-red-50 rounded-lg text-center">
              <p className="text-red-600 font-medium">Something went wrong</p>
              <p className="text-sm text-red-500 mt-1">{error.message}</p>
              <button
                onClick={resetErrorBoundary}
                className="mt-4 px-4 py-2 bg-red-600 text-white rounded-md hover:bg-red-700"
              >
                Try again
              </button>
            </div>
          )}
        >
          {children}
        </ErrorBoundary>
      )}
    </QueryErrorResetBoundary>
  );
}
```
