## üîê Security Guidelines

<Callout type="error">Never share sensitive information with AI tools.</Callout>

### üö´ What NOT to Share

- API keys and secrets
- Database credentials
- User personal data
- Internal URLs and endpoints
- Proprietary business logic
- Customer information

### ‚úÖ Safe Practices

```tsx
// ‚ùå Bad - Contains real credentials
const API_KEY = "sk-real-api-key-12345";

// ‚úÖ Good - Use placeholder
const API_KEY = process.env.API_KEY; // "your-api-key-here"
```

### üßπ Code Sanitization

Before sharing code with AI:

1. Replace real API keys with placeholders
2. Remove user data and PII
3. Anonymize business-specific logic if needed
4. Remove internal URLs

## ‚úÖ Quality Checklist

Before committing AI-generated code:

- [ ] Code compiles without errors
- [ ] All tests pass
- [ ] No linting warnings
- [ ] Types are correct and complete
- [ ] Follows project conventions
- [ ] Accessible (ARIA, keyboard nav)
- [ ] No hardcoded values
- [ ] Error handling included
- [ ] Security reviewed
- [ ] Performance acceptable

## ‚ö†Ô∏è Common Pitfalls

| Pitfall            | Solution                                               |
| ------------------ | ------------------------------------------------------ |
| Outdated syntax    | Specify versions: "Using React 19..."                  |
| Wrong patterns     | Reference existing code: "Match style in UserCard.tsx" |
| Missing edge cases | Ask: "What edge cases should I handle?"                |
| Over-engineering   | Ask for "simple, minimal implementation"               |
| No error handling  | Explicitly request error handling                      |
